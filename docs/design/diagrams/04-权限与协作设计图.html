一、权限层级架构

改造思路：从简单的 createdBy + isPublic 二元控制，升级为「系统角色 + 知识库成员 + 资源级权限」三层模型。AccessControlService 统一封装所有权限判断逻辑。
🏛️
第一层：系统角色
Admin User
全局管理权限
👥
第二层：知识库成员 新增
ADMIN EDITOR VIEWER
知识库级细粒度权限
📄
第三层：资源操作 新增
文档审批 · 评论权限
知识库设置 · 成员管理
🛡️
AccessControlService — 统一权限层
canAccessKb() getKbRole() canEditDocument() canManageKb() buildKbAccessFilter() buildDocAccessFilter()
替代各 Service 中分散的 OR 过滤逻辑
