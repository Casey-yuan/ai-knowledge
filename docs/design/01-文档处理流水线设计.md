## 设计文档：文档处理流水线增强

版本：v1.0 | 日期：2026-06-11

---

### 一、现状分析

当前 `IngestionProcessor`（ingestion.processor.ts）存在以下问题：

1. **文件解析缺失**：所有 FILE 类型文档统一使用 `fs.readFileSync(filePath, 'utf-8')` 读取，PDF、Word、HTML 虽然依赖已安装（pdf-parse、mammoth、cheerio 在 package.json 中），但解析逻辑未实现。
2. **分块逻辑不一致**：`ChunkingService` 提供了递归字符分割算法，但 `IngestionProcessor` 使用自己的 `chunkContent` 私有方法，两者未统一。
3. **分块策略单一**：只有按字符递归分割一种策略，无法适应不同文档类型的结构特征。
4. **无父子分块**：检索命中子块时无法返回上下文父块，回答完整性受限。
5. **分块参数硬编码**：chunkSize=500、overlap=50 写死在代码中，无法按知识库或文档类型配置。

---

### 二、目标

1. 实现 PDF、Word（docx）、HTML 的真实文件解析
2. 统一分块调用入口，消除重复逻辑
3. 引入多种分块策略（递归字符、结构化、语义），支持按文档类型自适应选择
4. 实现父子分块机制，检索时返回上下文
5. 知识库级别可配置分块参数

### 四、模块设计

#### 4.1 文件解析器（Parser Registry）

采用策略模式，根据 mimeType 分发到对应的解析器。

```
src/modules/ingestion/parsers/
├── parser.interface.ts       // 解析器接口
├── parser.registry.ts        // 解析器注册表
├── text.parser.ts            // 纯文本 / Markdown
├── pdf.parser.ts             // PDF（pdf-parse）
├── word.parser.ts            // Word（mammoth）
└── html.parser.ts            // HTML（cheerio）
```

**接口定义**：

```typescript
// parser.interface.ts
export interface ParsedContent {
  text: string;                   // 提取的纯文本
  sections?: ParsedSection[];     // 结构化分段（用于结构化分块）
  metadata?: Record<string, any>; // 文档元数据（页数、作者等）
}

export interface ParsedSection {
  type: 'heading' | 'paragraph' | 'code' | 'table' | 'list';
  content: string;
  level?: number;     // heading 层级 (1-6)
  language?: string;  // code 的语言
}

export interface DocumentParser {
  supportedMimeTypes: string[];
  parse(filePath: string, buffer?: Buffer): Promise<ParsedContent>;
}
```

**注册表**：

```typescript
// parser.registry.ts
@Injectable()
export class ParserRegistry {
  private parsers = new Map<string, DocumentParser>();

  constructor(
    private textParser: TextParser,
    private pdfParser: PdfParser,
    private wordParser: WordParser,
    private htmlParser: HtmlParser,
  ) {
    this.register(textParser);
    this.register(pdfParser);
    this.register(wordParser);
    this.register(htmlParser);
  }

  private register(parser: DocumentParser) {
    for (const mime of parser.supportedMimeTypes) {
      this.parsers.set(mime, parser);
    }
  }

  getParser(mimeType: string): DocumentParser {
    return this.parsers.get(mimeType) || this.textParser;
  }
}
```

**各解析器要点**：

| 解析器 | 依赖 | 核心逻辑 |
|---|---|---|
| TextParser | 无 | 直接读取 UTF-8，按 `\n` 识别段落 |
| PdfParser | pdf-parse | `pdf-parse(buffer)` 提取文本，按页拆分 sections |
| WordParser | mammoth | `mammoth.extractRawText({ buffer })` 转纯文本 |
| HtmlParser | cheerio | 提取 `<body>` 正文，去除 `<script>`/`<style>`/`<nav>` 噪声 |

#### 4.2 分块策略（Chunking Strategy）

```
src/modules/chunking/
├── chunking.service.ts           // 统一入口（已有，需重构）
├── strategies/
│   ├── chunking-strategy.interface.ts
│   ├── recursive.strategy.ts     // 现有逻辑提取
│   ├── structure.strategy.ts     // 基于文档结构的分块
│   └── semantic.strategy.ts      // 基于语义相似度的分块
```

**接口**：

```typescript
export interface ChunkingStrategy {
  name: string;
  chunk(content: ParsedContent, options: ChunkOptions): ChunkResult[];
}

export interface ChunkOptions {
  chunkSize: number;
  overlap: number;
}

export interface ChunkResult {
  content: string;
  type: string;         // text, heading, code, table
  level: number;        // 层级
  parentId?: string;    // 父块索引（在结果数组中的索引）
  startOffset: number;
  endOffset: number;
}
```

**三种策略说明**：

| 策略 | 适用场景 | 核心逻辑 |
|---|---|---|
| RecursiveStrategy | 通用文档、纯文本 | 现有递归字符分割算法，按 `\n\n` > `\n` > `。` > `.` 等优先级切分 |
| StructureStrategy | Markdown、HTML、PDF | 根据 `ParsedSection` 的 heading 层级构建父子树，每个 section 内再按 chunkSize 切分 |
| SemanticStrategy | 高精度场景 | 先按句子分割，逐句计算 embedding，当相邻句子相似度低于阈值时切断 |

#### 4.3 父子分块检索

在 `RetrievalService` 中增加父块上下文返回：

```typescript
async searchWithParentContext(
  kbId: string, query: string, topK: number, threshold: number
) {
  // 1. 先检索子块
  const childChunks = await this.hybridSearch(kbId, query, topK, threshold);

  // 2. 收集所有 parentId
  const parentIds = [...new Set(
    childChunks.filter(c => c.parentId).map(c => c.parentId)
  )];

  // 3. 批量查询父块
  const parents = parentIds.length > 0
    ? await this.prisma.chunk.findMany({ where: { id: { in: parentIds } } })
    : [];

  // 4. 返回子块 + 父块上下文
  return childChunks.map(child => ({
    ...child,
    parentContent: parents.find(p => p.id === child.parentId)?.content || null,
  }));
}
```

### 五、IngestionProcessor 重构

重构后的处理流程：

```
processDocument(job):
  1. 查询文档和所属知识库（获取分块配置）
  2. 状态 → PARSING
  3. 根据 mimeType 从 ParserRegistry 获取解析器
  4. 调用 parser.parse() 得到 ParsedContent
  5. 状态 → CHUNKING
  6. 删除旧 chunks
  7. 根据 kb.chunkStrategy 从 ChunkingService 获取策略
  8. 调用 strategy.chunk(parsedContent, { chunkSize, overlap })
  9. 创建 Chunk 记录（含 parentId、level、offset、type）
  10. 状态 → EMBEDDING
  11. 批量生成 embedding（仅对 level=1 的子块，或 level=0 的独立块）
  12. 状态 → COMPLETED
```
